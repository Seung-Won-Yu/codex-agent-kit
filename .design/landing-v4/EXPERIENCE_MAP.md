# Landing v4 Experience Map

## Page flow

1. **Promise**
   - Brand statement, install CTA, skill catalogue CTA
   - Precise animated route lines establish the system metaphor
2. **Try the system**
   - Choose a rough-request scenario
   - Watch Interpret → Permission → Route → Verify advance
   - Inspect the normalized result and selected responsibility slots
3. **Understand the contract**
   - Scroll through four chapters
   - A sticky orchestration map changes with the current chapter
4. **Inspect the implementation**
   - Agent roles
   - Searchable personal skills
   - Connected plugins
5. **Install**
   - Copy installation and verification commands

## Interaction states

- Demo: idle, processing, complete
- Scenario: selected and unselected
- Story: intent, permission, routing, verification
- Theme: light, changing, dark
- Search: closed, open, no results, results
- Skill catalogue: preview, filtered, expanded
- Copy: idle, copied, failed

## Responsive behavior

- Desktop: sticky two-column orchestration story
- Tablet: reduced visual scale with the same chapter progression
- Mobile: orchestration visual stays in normal flow; chapter controls become a compact horizontal selector
- Reduced motion: all content is immediately visible and state changes happen without travel or looping animation

# Landing v4 Design Brief

## Product job

Show the personal Codex setup as an operating system that can be understood and tried, not as a long catalogue of configuration files.

## Primary user

A developer who lands on the repository from GitHub and wants to know, within one minute:

1. what this setup changes,
2. how it interprets brief requests,
3. how it preserves authority and selects skills,
4. whether the setup can be installed and trusted.

## Success signal

- The first viewport communicates the promise as one complete scene; the interactive demonstration begins immediately after it without a clipped preview.
- Scrolling through the system story visibly advances intent, permission, routing, and verification.
- Light and dark themes have equal visual depth and motion.
- The page remains useful with JavaScript disabled and fully readable with reduced motion.
- No horizontal overflow at 320, 768, 1024, or 1440 px.

## Scope

- Hero and product-demo art direction
- Interactive processing timeline
- Scroll-led orchestration story
- Editorial refinement of agent, skill, plugin, and installation sections
- Theme, keyboard, responsive, accessibility, and motion verification

## Non-scope

- Changes to the actual Codex configuration, skills, agents, or installation behavior
- New framework or runtime dependency
- Analytics, account features, or backend services

## Direction

Precise editorial typography, technical instrumentation, and restrained signal color. Avoid decorative glass cards, purple AI gradients, generic floating orbs, and motion without product meaning.

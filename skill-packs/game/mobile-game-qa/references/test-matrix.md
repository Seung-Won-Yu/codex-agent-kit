# Mobile Game QA Test Matrix

Use this for browser or app verification.

## Viewports

- Small phone portrait: 360 x 740
- Modern phone portrait: 390 x 844
- Large phone portrait: 430 x 932
- Small landscape: 740 x 360
- Tablet if relevant

## Core Flows

- Launch and loading state
- First tap / first interaction
- Gameplay start
- Core action repeated 10 times
- Win state
- Fail state
- Reward claim
- Upgrade or progression
- Pause/resume
- Restart
- Settings / mute

## Checks

- Text does not overflow buttons, HUD, or modals.
- HUD does not cover play targets.
- Touch targets are not too small or too close.
- Disabled states are visually distinct.
- Loading and error states exist.
- Motion does not hide important state changes.
- Audio has mute control if audio exists.
- Game remains playable without color-only cues.
- Browser UI or safe areas do not hide controls.

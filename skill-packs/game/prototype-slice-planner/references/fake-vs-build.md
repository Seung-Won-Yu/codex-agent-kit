# Fake vs Build Guide

Use this to keep a prototype focused.

## Usually Build

- The core input.
- The immediate feedback loop.
- The main state changes the player must understand.
- One win condition.
- One failure or restart condition.
- Minimal HUD needed to play.

## Usually Fake

- Account login.
- Persistent backend.
- Full economy.
- Full content library.
- Shop purchases.
- Ads or monetization.
- Tutorial system.
- Live events.
- Social graph.
- Settings beyond mute and reset.

## Depends

- Progression: build one upgrade if motivation depends on it.
- Character choice: build two options only if comparison is the hook.
- Inventory: build fixed slots if collection is the hook.
- Procedural generation: fake with hand-authored content unless randomness is the core risk.
- Audio: use simple placeholders if feel is part of the hypothesis.

## Decision Question

Will building this change the answer to the prototype hypothesis?

If no, fake it.

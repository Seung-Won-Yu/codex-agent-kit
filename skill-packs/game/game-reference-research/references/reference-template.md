# Skill Reference Template

Use this template when turning research into a compact `references/*.md` file.

```markdown
# {Topic} Reference

Use this when {trigger or task}.

## What This Helps With

- {decision/task}
- {decision/task}
- {decision/task}

## Core Patterns

### {Pattern Name}

- Use when:
- How it works:
- Implementation notes:
- Risks:

## Checklist

- [ ] {testable check}
- [ ] {testable check}
- [ ] {testable check}

## Good Defaults

- {default}
- {default}
- {default}

## Avoid

- {anti-pattern}
- {anti-pattern}
- {anti-pattern}

## Example Prompts

- "Use ${skill-name} to ..."
- "Review this ... using ..."

## Sources

- [{Source title}]({url}) - why it matters
- [{Source title}]({url}) - why it matters
```

## Writing Rules

- Keep it short enough to load during normal work.
- Prefer concrete thresholds, examples, and decision rules.
- Do not include broad background unless it changes execution.
- Include links, but make the reference usable even if offline later.
- Put volatile facts behind dated source notes.

Why This Exists

많은 기능보다 중요한 것은, 매번 덜 헤매는 기본값입니다.

Codex는 스킬, 플러그인, MCP, project rule, browser automation, 문서 생성 도구까지 붙일 수 있어서 강력합니다. 하지만 그대로 늘려두면 매 요청마다 "어떤 역할을 써야 하지?"라는 분기가 많아집니다. 이 kit는 그 분기를 줄이기 위해 만들었습니다.

현재 세팅의 핵심은 "작게 라우팅하고 빨리 실행한다"입니다. 명확한 구현, 문서, 리서치, UI 수정은 바로 관련 파일을 읽고 작업합니다. 요청이 애매하거나 여러 분야가 섞였을 때만 messy-request-interpreter와 skill-router를 거칩니다.

작업 속도

역할 놀이와 과한 분류를 줄여 실제 편집, 검증, 배포에 더 빨리 들어갑니다.

결과 일관성

요청이 짧아도 항상 파일 읽기, 근거 확인, 집중 검증, 짧은 handoff 순서를 유지합니다.

공유 가능성

인증값과 로컬 상태를 제외하고, 규칙과 reusable skills만 repo에 남겨 다른 사람이 참고할 수 있습니다.

안전한 경계

세션, SQLite state, auth, plugin cache, private path는 의도적으로 공유 대상에서 제외합니다.

Setup Layers

현재 Codex 설정은 여섯 개 레이어로 나눠 관리합니다.

01

Global Agent Rules

AGENTS.md는 모든 작업의 기본 태도입니다. 명확하면 바로 실행하고, 애매하면 최소한으로 해석하고, 검증 가능한 형태로 마무리하라는 규칙을 둡니다.

왜 씀
매번 새로 지시하지 않아도 Codex가 같은 기준으로 움직이게 하기 위해서입니다.
장점
요청이 짧아도 실행 방식, 언어, 검증, 커뮤니케이션 톤이 흔들리지 않습니다.
02

Routing Reference And Playbooks

agents/routing.md와 agents/playbooks/는 프론트엔드, 백엔드, 디자인, 문서/리서치처럼 큰 갈래가 헷갈릴 때만 참조하는 얇은 지도입니다.

왜 씀
모든 요청에 무거운 워크플로를 태우지 않고, 필요한 순간에만 판단 근거를 꺼내기 위해서입니다.
장점
작은 수정은 빠르게, 복합 작업은 놓치는 영역 없이 처리할 수 있습니다.
03

Lean Skill Profile

config/lean-skills.txt에는 현재 로컬에서 유지할 대표 스킬 103개가 들어 있습니다. 모든 스킬을 다 켜두는 대신, 자주 쓰는 작업군별 대표 스킬만 남깁니다.

왜 씀
스킬이 너무 많으면 선택지가 늘어 응답이 무거워지고 라우팅이 산만해집니다.
장점
디자인, 구현, Supabase, GitHub, 문서, 리서치, 배포를 커버하면서도 운영 부담을 낮춥니다.
04

Curated Skills

이 repo의 skills/에는 직접 쓰는 커스텀/큐레이션 스킬 31개를 저장합니다. 전체 로컬 스킬셋은 더 넓지만, 공유 가능한 핵심만 남깁니다.

왜 씀
개인 workflow를 재사용 가능한 단위로 고정하고, 다른 환경에도 쉽게 옮기기 위해서입니다.
장점
프론트엔드 품질, 게임 기획, 이미지 생성, Supabase, Open Design artifact 작업이 반복 가능해집니다.
05

Redacted Config Sample

config/codex.config.sample.toml은 실제 설정의 형태만 보여줍니다. model, reasoning, feature flag, plugin, MCP 구조는 남기고, 개인 경로와 인증 상태는 제거합니다.

왜 씀
다른 사람이 구조를 이해하되, 로컬 경로나 인증값을 따라 복사하지 않게 하기 위해서입니다.
장점
세팅 철학은 공유하고, 민감한 실행 환경은 보호할 수 있습니다.
06

Safety Boundary

이 kit는 auth.json, SQLite state, sessions, logs, browser session, plugin runtime cache, private attachments를 저장하지 않습니다.

왜 씀
Codex home은 그대로 공유하면 민감한 상태와 인증 흔적이 섞일 수 있습니다.
장점
공개 repo로 운영해도 설정 철학과 재사용 가능한 파일만 남습니다.

Operating Flow

요청의 선명도에 따라 다른 길을 탑니다.

명확한 단일 작업

예: "이 버그 고쳐줘", "README 정리해줘", "이 페이지 모바일 깨져".

read files -> edit -> verify -> concise final 효과

불필요한 역할 선택 없이 바로 실제 작업으로 들어갑니다.

애매한 혼합 요청

예: "대충 알아서 멋있게", "내 세팅 좀 정리해서 공유하고 싶어".

messy-request-interpreter -> skill-router -> execution 효과

사용자의 의도를 작게 해석한 뒤, 산만한 작업을 실행 가능한 순서로 바꿉니다.

프론트엔드/디자인

기존 UI면 먼저 구조와 화면을 보고, 새 페이지면 목적과 독자를 기준으로 구성합니다.

frontend-design -> implementation -> Playwright check 효과

시각적 완성도와 실제 반응형 안정성을 함께 봅니다.

백엔드/Supabase

Auth, RLS, schema, query, migration처럼 위험도가 높은 작업은 보안과 데이터 경계를 같이 봅니다.

supabase -> postgres best practices -> focused verification 효과

권한, 소유권, 인덱스, migration safety를 놓칠 가능성을 줄입니다.

Codex Agent Kit

Seung-Won Yu가 현재 로컬 workspace에서 실제로 사용하는 Codex 설정을 공유 가능한 형태로 정리한 repository입니다. 이 repo는 ~/.codex 전체 백업이 아니라, 인증값과 로컬 state를 제외한 portable layer만 남깁니다.

공개 페이지 https://seung-won-yu.github.io/codex-agent-kit/

이 세팅의 목적

Codex는 skills, plugins, MCP, browser automation, 문서/스프레드시트/PDF 도구까지 붙일 수 있어 강력합니다. 하지만 모든 것을 한 번에 켜두면 매 요청마다 어떤 역할과 workflow를 선택해야 하는지 분기가 많아집니다.

  • 명확한 요청은 바로 실행합니다.
  • 애매한 요청만 작게 해석하고 라우팅합니다.
  • 스킬은 대표 workflow 중심으로 lean하게 유지합니다.
  • 실제 작업은 파일 읽기, 수정, 검증, 짧은 handoff까지 끝냅니다.
  • 공유 repo에는 규칙과 reusable skills만 남기고 민감한 로컬 상태는 제외합니다.

AGENTS.md

Global Agent Rules

이 파일은 Codex가 현재 workspace에서 어떤 태도로 움직일지 정하는 최상위 규칙입니다. 핵심은 lightest workflow, direct execution, Korean-first communication, focused verification입니다.

Core Behavior

  • 명확한 single-domain task는 바로 실행합니다.
  • role selection이나 skill-router ceremony는 ambiguity가 실제 결과에 영향을 줄 때만 씁니다.
  • classification보다 execution을 우선합니다.
  • 사용자가 한국어를 쓰면 code나 artifact convention을 제외하고 한국어로 답합니다.

Fuzzy Intake

messy-request-interpreter -> skill-router -> execution

이 route는 원하는 산출물, owner role, implementation path가 불명확할 때만 사용합니다. 단순 작업에는 과한 routing을 넣지 않습니다.

Execution Discipline

  • guess보다 evidence를 우선합니다.
  • non-trivial change 전에는 관련 파일을 먼저 읽습니다.
  • 사용자가 만든 변경은 되돌리지 않습니다.
  • 가능하면 focused verification을 실행합니다.
  • routine reply는 짧게, user-facing artifact는 정상적인 전문 문체로 작성합니다.
docs/architecture.md

Architecture

이 kit는 raw dotfiles가 아니라 여섯 개의 공유 가능한 레이어로 나뉩니다.

01

Routing Layer

AGENTS.md가 명확한 요청은 직접 실행하고, 애매한 요청만 fuzzy intake로 넘깁니다.

02

Agent Reference Layer

agents/routing.md와 playbooks가 frontend, backend, docs/research 기준을 제공합니다.

03

Skill Layer

skills/는 반복 가능한 workflow입니다. 각 skill은 trigger, process, guardrail을 담습니다.

04

Local Config Layer

config/codex.config.sample.toml은 feature/plugin/MCP 형태만 보여주고 민감값은 제외합니다.

05

Documentation Site Layer

index.html과 assets/가 GitHub Pages 공유 화면을 담당합니다.

06

Third-Party Import Layer

third_party/와 od- prefix로 imported skills와 license 경계를 분리합니다.

공유하지 않는 것

  • auth.json, sessions, logs, memories, state SQLite
  • browser sessions, plugin caches, runtime bundles
  • private attachments, generated images with private data

Lean Skill Profile

이 profile은 Codex를 유용하게 유지하면서도 모든 installed skill을 한 번에 노출하지 않기 위한 기준입니다.

Goal

  • 약 100개의 대표 skills를 유지합니다.
  • design, frontend, game, image generation, GitHub, Supabase, docs, research, implementation을 커버합니다.
  • active skill set을 작게 유지해 routing을 빠르게 만듭니다.
  • senior developer 기준은 별도 persona가 아니라 shared lens로 흡수합니다.
  • caveman은 routine communication에만 쓰고 artifact 문체는 보존합니다.

docs/security-checklist.md

Security Checklist

공개 repo로 올리기 전에 아래 항목을 확인합니다.

No auth.json No .env files No API keys or OAuth tokens No SQLite state or sessions No browser session files No private screenshots No plugin runtime cache No private customer data

Suggested scan

rg -n "(api[_-]?key|secret|token|password|Bearer|sk-[A-Za-z0-9]|gho_|github_pat|refresh_token|PRIVATE KEY)" .

문서 예시 때문에 일부 match가 나올 수 있으니 push 전에 직접 확인합니다.

Open Design Imports

이 kit는 Open Design ecosystem 전체를 복제하지 않고, Codex workflow에 실제로 도움이 되는 일부 skill만 od- prefix로 가져옵니다. 이렇게 하면 local skill과 이름이 충돌하지 않고 출처도 구분됩니다.

왜 가져왔나

  • dashboard와 visual report 같은 artifact workflow를 보강합니다.
  • research synthesis를 decision artifact로 바꾸는 경로를 추가합니다.
  • score-based design review gate를 넣어 UI 품질을 점검합니다.
  • D3/data visualization, hand-drawn diagrams, motion frames를 커버합니다.
  • DESIGN.md를 design-system source of truth로 다룰 수 있게 합니다.

대표 imports

od-design-md

visual rules, tokens, product design direction을 DESIGN.md로 관리합니다.

od-platform-design

Apple HIG, Material, accessibility 기준을 cross-platform design에 반영합니다.

od-plan-design-review

shipping 전 senior design review gate로 사용합니다.

od-d3-visualization

D3 chart와 interactive visualization guidance를 제공합니다.

od-data-report

structured data를 polished visual report page로 바꿉니다.

od-live-dashboard

KPI, activity feed, tables를 포함한 dashboard artifact를 만듭니다.

Import Policy

  • imported skill은 od- prefix를 유지합니다.
  • license는 third_party/open-design/LICENSE에 보관합니다.
  • 필요한 skill만 추가하고, 전체 catalog를 기본 import하지 않습니다.

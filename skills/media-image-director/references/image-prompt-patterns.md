# Image Prompt Patterns

## Poster Or Thumbnail

```text
Create a vertical poster image for [project/product].
Purpose: [GitHub README / launch post / thumbnail].
Composition: one clear central subject, large negative space for title, strong foreground/background separation.
Visual style: [specific style].
Palette and lighting: [specific colors].
Text: "[exact short title]" only.
Quality guardrails: readable title, no extra text, no fake logos, no clutter, no distorted UI.
```

## Product Or App Mockup

```text
Create a clean product showcase image for [app/project].
Purpose: portfolio and README preview.
Composition: [phone/laptop/browser] screen showing the actual product idea, angled slightly, with a simple background.
Visual style: polished but inspectable, product-first, realistic UI density.
Palette and lighting: [colors].
Text: no tiny fake UI text except short visible labels.
Quality guardrails: crisp screen, readable layout, no random extra app names, no distorted controls.
```

## Game Concept Image

```text
Create a mobile casual game concept image for [game idea].
Purpose: visual direction and prototype reference.
Composition: [game board / reward popup / character / HUD] as the focal point.
Visual style: [cute casual / arcade / pixel / fantasy / cozy].
Palette and lighting: [colors].
Key details: clear feedback, readable rewards, touch-friendly icons, one obvious next action.
Text: no readable text unless specified.
Quality guardrails: no cluttered HUD, no tiny unreadable labels, no random characters, no broken hands.
```

## Style Variation

```text
Create a variation of the provided image.
Preserve: [subject/composition/colors/identity].
Change: [style/background/lighting/mood/crop].
Purpose: [use case].
Quality guardrails: keep the subject recognizable, avoid adding unrelated objects, keep edges clean.
```

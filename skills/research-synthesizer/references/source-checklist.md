# Source Checklist

Use this checklist when evaluating external information.

## Trust Signals

- Official organization or maintainer
- Clear license
- Current docs or changelog
- Install instructions that match the target platform
- Minimal hidden dependencies
- Examples that can be verified

## Red Flags

- No source code or no visible `SKILL.md` for a skill
- Broad claims without examples
- Scripts that require secrets without explaining why
- Large permission surface for a small task
- Abandoned repo or stale docs for fast-moving software
- Marketplace mirror with no link to the primary repo

## Final Citation Rule

Always include links to the sources actually used. Do not cite sources that were only glanced at and did not inform the answer.

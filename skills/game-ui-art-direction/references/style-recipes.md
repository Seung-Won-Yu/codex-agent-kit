# Game UI Style Recipes

Use these as starting points. Adapt to the product, do not stack multiple recipes at once.

## Cozy Casual

- Shapes: soft rounded controls, simple icons, friendly spacing.
- Palette: fresh greens, warm yellows, sky accents, off-white surfaces.
- Type: rounded sans, medium weight, high readability.
- Motion: bounce on reward, gentle progress fills.
- Avoid: tiny labels, heavy chrome, harsh contrast.

## Arcade Neon

- Shapes: strong outlines, high-contrast panels, sharp meters.
- Palette: black or deep navy base, cyan, magenta, lime, white.
- Type: condensed or geometric display for headers, readable sans for body.
- Motion: quick flashes, scan lines, punchy transitions.
- Avoid: glowing all text, low-contrast neon on neon.

## Fantasy RPG

- Shapes: carved panels, framed inventory slots, gem-like resources.
- Palette: parchment or deep stone base, gold accents, rarity colors.
- Type: readable serif/display only for headers; body must stay plain.
- Motion: shimmer for rarity, soft particles on reward.
- Avoid: unreadable ornate fonts, too many borders.

## Cute Collection

- Shapes: pill buttons, sticker-like items, large item thumbnails.
- Palette: candy colors balanced with neutral surfaces.
- Type: round sans, clear numerals for resources.
- Motion: pop-in rewards, item wiggle on selection.
- Avoid: over-saturated full-screen backgrounds that compete with items.

## Minimal Premium

- Shapes: restrained panels, precise spacing, subtle depth.
- Palette: dark neutral or light neutral with one strong accent.
- Type: clean sans, generous line height.
- Motion: short fades, measured state transitions.
- Avoid: empty SaaS dashboard feeling; keep game-specific feedback and reward moments.

## Pixel Retro

- Shapes: blocky buttons, grid slots, hard shadows.
- Palette: limited palette with strong contrast.
- Type: pixel font for display only; readable fallback for body.
- Motion: stepped animation, tile flips, score ticks.
- Avoid: using pixel fonts for long text or small mobile labels.

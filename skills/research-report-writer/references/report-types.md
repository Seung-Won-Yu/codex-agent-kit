# Research Report Types

Use this table to choose the report shape before researching.

| User asks for | Best artifact | Use when | Pair with |
|---|---|---|---|
| 리서칭보고서, 조사보고서 | General Research Report | Topic needs source-backed overview and recommendation | `research-synthesizer`, `research-report-writer` |
| 기술조사, 라이브러리 비교 | Technical Discovery Report | Need docs, APIs, GitHub, package or tool comparison | `research-synthesizer`, `technical-writer` |
| 벤치마킹 | Benchmarking Report | Need compare products, apps, competitors, features, UX, pricing | `research-synthesizer`, `research-report-writer` |
| 경쟁사분석 | Competitor Analysis | Need competitor positioning, features, pricing, GTM, strengths | `research-synthesizer`, `product-strategy` |
| 시장조사 | Market Research Report | Need market size, audience, trends, segments, opportunity | `research-synthesizer`, `product-strategy` |
| TAM/SAM/SOM | Market Sizing Report | Need investor/business market estimate | `research-synthesizer`, `technical-writer` |
| 트렌드 리포트 | Trend Report | Need emerging signals, recent shifts, weak signals | `research-synthesizer`, `research-report-writer` |
| 논문/자료 요약 | Literature / Source Review | Need summarize several papers/docs/sources | `research-synthesizer`, `technical-writer` |
| 의사결정 보고서 | Decision Memo | Need choose between options with tradeoffs | `research-synthesizer`, `planning-document-writer` |
| 레퍼런스 조사 | Reference Map | Need reusable source list, examples, patterns, GitHub candidates | `research-synthesizer`, `game-reference-research` when game-related |
| 유저 리서치 | User Research Synthesis | Need interviews, surveys, VOC, pain points, jobs-to-be-done | `research-report-writer`, `product-strategy` |
| 광고/마케팅 리서치 | Ads / Marketing Research | Need competitor ads, messages, hooks, positioning | `research-synthesizer`, `product-strategy` |

## Default Templates

### General Research Report

1. Executive summary
2. Research question
3. Scope and exclusions
4. Key findings
5. Evidence table
6. Analysis
7. Recommendation
8. Risks and unknowns
9. Sources

### Technical Discovery Report

1. Decision to make
2. Candidate options
3. Comparison matrix
4. Official docs / primary sources
5. Integration cost
6. Maintenance and adoption
7. Security/licensing risks
8. Recommended option
9. Implementation notes

### Benchmarking Report

1. Benchmark objective
2. Candidates
3. Comparison criteria
4. Feature / UX / pricing matrix
5. Notable patterns
6. Gaps and opportunities
7. Recommended direction
8. Source links

### Market Research Report

1. Market definition
2. Target segment
3. Market size / growth
4. Customer needs
5. Competitive landscape
6. Trends and drivers
7. Opportunity and risks
8. Recommended positioning
9. Sources and assumptions

### Decision Memo

1. Recommendation
2. Context
3. Options considered
4. Decision criteria
5. Evidence
6. Tradeoffs
7. Risks
8. Next steps

# Mobile Game GDD Checklist

Use this when drafting or reviewing a mobile game design document.

## Required Sections

- One-sentence pitch
- Target player and play context
- Player fantasy
- Genre and nearest references
- Core loop
- Controls and input model
- First-time user experience
- Session structure
- Win, lose, and partial-success states
- Progression and unlocks
- Reward types and reward cadence
- Economy sources and sinks
- Content structure: levels, stages, enemies, items, cards, characters, or puzzles
- UI screens and HUD
- Audio, haptics, and feedback moments
- Accessibility and localization constraints
- Analytics events
- Prototype scope
- Open risks and playtest questions

## Review Prompts

- Can a player understand the goal without reading a long tutorial?
- Is the first success reachable within the first minute?
- Does every repeated action change a visible state?
- Is there a meaningful decision, or only repeated tapping?
- Does the progression system create a reason to return without punishing missed days too hard?
- Is the economy understandable without a spreadsheet?
- What can be tested with placeholder art?
- What would a bored player do first?

## Red Flags

- The core loop is described as a list of features instead of player actions.
- The first session depends on complex meta systems.
- Rewards exist, but no interesting choice follows them.
- The game uses multiple currencies before the player has learned why one matters.
- The design assumes players will read tutorials.
- Retention comes only from timers, streak pressure, or notifications.

# Design And Prototype Playbook

Use this only when the user asks for fuzzy product/design work, visual direction, a prototype, or "something like this."

## Counterproposal

Before execution, briefly map the request:

```text
내 해석: ...
추천 경로: ...
대안: ...
진행: ...
```

## Mode Map

- Fast visual artifact or concept: visual/design skill such as `$claude-design` if installed.
- Working local app/prototype: `$product-frontend-engineer`.
- Full design workflow: `$design-flow`.
- Existing UI cleanup: `$ux-enhancer`.
- Generated image, poster, thumbnail, app/game visual, product mockup, or prompt iteration: `$media-image-director`.
- Structured Open Design artifact: exact `od-*` skill.

## Execution

- If one path is clearly best and low-risk, state the recommendation and proceed.
- Build the usable experience first, not a marketing page, unless the user asks for marketing.
- Pair frontend/game/visual work with inspection or browser verification when feasible.

# Documents And Research Quality Checklist

Use only for substantial documents, reports, specs, runbooks, SOPs, release notes, or current-fact research. Preserve the request's intent and effect ceiling; research or a proposal does not authorize file or external changes.

- Match the artifact, audience, decision, and level of formality to the user's actual use case.
- Use the narrowest content-owner skill; add a document/PDF/slides adapter only when that output format is required.
- Prefer primary and official sources for current technical facts; browse when facts may have changed.
- Keep sourced facts, inference, and recommendations distinguishable.
- Preserve exact dates, terms, paths, commands, API names, and version constraints.
- Verify links, commands, citations, and artifact rendering where relevant.
- Keep routine summaries concise; use formal structure only when the deliverable calls for it.

# Docs And Research Playbook

Use this only for substantial documents, reports, specs, runbooks, SOPs, release notes, or current-fact research.

## Document Routing

- Planning document, PRD, requirements, feature spec, screen spec, GDD, DB docs, SOP, or runbook: `$planning-document-writer`, then exact artifact skill if needed.
- Technical documentation: `$technical-writer`.
- API documentation: `$api-documentation-writer`.
- Operations docs: `$runbook-generator` or `$sop-builder`.
- Release/history: `$release-notes`, `$changelog-generator`, or `$handoff`.

## Research Routing

- Research-backed report, comparison, benchmark, market/technical scan, or decision memo: `$research-report-writer` or `$research-synthesizer`.
- Use web research for current facts, pricing, law, schedules, product details, or anything likely to have changed.
- Cite sources and separate sourced facts from inference.

## Output

- Match the artifact to the user's use case.
- Keep routine summaries concise.
- Preserve exact terms, dates, paths, commands, and API names.
